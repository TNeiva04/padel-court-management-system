## Single Page Application (SPA)

A aplicação inclui uma interface web desenvolvida como uma **Single Page Application (SPA)**, permitindo uma interação fluida com os dados da API, sem necessidade de recarregar a página. Esta interface é inteiramente desenvolvida com **HTML, CSS e JavaScript**.

### Arquitetura e Organização do Código

O frontend está localizado na pasta 'static-content/', organizado modularmente com os seguintes ficheiros principais:
- **`index.html`**  
  Ficheiro principal que define a estrutura da página. Contém links para os ficheiros JavaScript e CSS, e elementos fixos como os links de navegação.

- **`index.js`**  
  Responsável por inicializar a aplicação no evento `load`, configurar as rotas com os respetivos handlers, e tratar das mudanças de hash na URL (`hashchange`) para simular navegação.

- **`router.js`**  
  Módulo que implementa o sistema de routing da SPA. Permite definir rotas com parâmetros (por exemplo, `clubs/{cid}`), associá-las a funções handler, e identificar qual handler executar com base no caminho atual (`window.location.hash`).

- **`handlers/`**  
  Contém os módulos responsáveis por cada tipo de conteúdo, no qual cada tipo de recurso (clubes, courts, rentals, utilizadores) têm um handler dedicado que produz a comunicação com a API e a renderização do conteúdo nos elementos principais da página:
  - `homeHandlers.js`: Gerencia a renderização da página inicial, incluindo a apresentação de informações gerais e links para outras secções.
  - `clubHandlers.js`: Trata a listagem de clubes, a paginação, a visualização de clubes específicos, a pesquisa por nome e a criação de clubes.
  - `courtHandlers.js`: Responsável por mostrar os courts de um clube, os detalhes de cada court individual, a visualização de horários disponíveis e a listagem de rentals por data.
  - `rentalsHandlers.js`: Gerencia a visualização das rentals realizadas, criação, atualização e remoção de reservas, além da navegação entre rentals.
  - `userHandlers.js`: Exibe detalhes do utilizador autenticado ou de outros utilizadores, conforme a navegação.
  - `session.js`: Controla a autenticação, armazenamento e renovação do token, e a gestão da sessão do utilizador, ajudando na autenticação.



### Funcionalidades Avançadas Implementadas

Além das operações básicas de leitura (GET), a SPA foi enriquecida com funcionalidades que permitem ao utilizador interagir de forma mais completa com o sistema, incluindo criação, atualização e remoção de recursos, bem como buscas específicas e navegação dinâmica.

- **Pesquisa de Clubes por Nome**  
  O utilizador pode pesquisar clubes pelo nome na página inicial e na listagem de clubes. A pesquisa é feita enviando o termo como query parameter para o endpoint `/clubs?name=...`, e a lista é atualizada dinamicamente com paginação.

- **Visualização de Horas Disponíveis para um Court numa Data Específica**  
  Na página de detalhes de um court, o utilizador pode selecionar uma data para consultar as horas disponíveis para reserva naquele court, consumindo o endpoint `/clubs/{cid}/courts/{crid}/availablehours?date=...`.

- **Busca de Rentals por Data no Detalhe do Court**  
  É possível listar as rentals de um court num dia específico, através da seleção da data, com dados obtidos via `/clubs/{cid}/courts/{crid}/rentals?date=...`.

- **Criação de Clubes, Courts e Rentals**  
  Formulários foram implementados para criar novos clubes, courts e rentals diretamente na SPA. As requisições POST enviam os dados para os endpoints correspondentes, incluindo o token de autenticação no header. Após a criação, a interface é atualizada com mensagens de sucesso ou erro.

- **Atualização e Remoção de Rentals**  
  A SPA permite editar Rentals existentes, apresentando formulários pré-preenchidos para alteração de data, hora e duração, enviando requisições PUT para atualizar. Também é possível excluir reservas via DELETE, com confirmação do utilizador.

- **Gestão completa de sessões e autenticação por utilização de tokens, garantindo segurança nas operações.**

## Interação com a API

A SPA realiza chamadas à API utilizando `fetch()` e manipulando diretamente o DOM para apresentar os dados ao utilizador.  
As chamadas incluem o token de autenticação, quando necessário (`Authorization: Bearer <token>`), e tratam erros com mensagens adequadas.

### Operações GET

Permitem obter listas ou detalhes de recursos (clubes, courts, rentals, utilizadores),fazendo com que sejam fundamentais para a navegação e visualização de informações

**Exemplos:**

- `#clubs` → `GET /clubs`, com paginação (`limit`/`skip`).
- `#clubs/3` → `GET /clubs/3` para obter detalhes de um clube específico.

### Operações POST

Para criar novos recursos, a SPA apresenta formulários interativos que recolhem dados 
do utilizador e enviam requisições POST para a API.
**Exemplo:**

- Criação de um clube ou  rental via formulário → `POST /clubs` ou `POST /rentals`.

### Operações PUT

A SPA suporta a atualização de recursos existentes, permitindo 
ao utilizador editar reservas já feita
**Exemplo:**
Exemplo: Ao editar uma reserva, a SPA carrega um formulário pré-preenchido com os dados actuais, que o utilizador pode modificar.
A submissão do formulário envia um pedido PUT para a API, actualizando o recurso no backend.
sendo que a  interface reflecte as alterações após a confirmação do sucesso da operação.
- Formulário pré-preenchido com dados da reserva → `PUT /rentals/{id}`.

### Operações DELETE


Para remoção de recursos, a SPA oferece mecanismos para cancelar Rentals ou eliminar dados, com confirmação do utilizador .
**Exemplo:**

- Clique no botão "Eliminar" → `DELETE /rentals/{id}`.



### Exemplo de Funcionamento de uma Função em `handlers/`

Por exemplo, a função `getClubs` de `clubHandlers.js` é responsável por obter e mostrar a lista de clubes na interface da aplicação, fazendo uma chamada à API para buscar os dados e depois atualizar o conteúdo da página dinamicamente.

#### Como funciona, de forma geral:

- A função começa por definir quantos clubes quer mostrar por página (`pageSize`) e mantém o controlo da página atual (`currentPage`).

- Depois, define uma função interna (`fetchAndRender`) que:
  - Calcula quantos clubes deve "pular" (`skip`) para obter a página correta (útil para a realização da paginação).
  - Faz uma requisição à API para buscar os clubes dessa página.
  - Verifica se existe uma página seguinte para saber se deve ativar ou desativar o botão **Next**.
  - Cria elementos HTML dinamicamente para mostrar os nomes dos clubes como links clicáveis.
  - Adiciona botões **Previous** e **Next** para navegar entre as páginas, atualizando a lista conforme o utilizador clica.

- Por fim, a função principal chama `fetchAndRender` para carregar a primeira página de clubes automaticamente.

- Em resumo: esta função, bem como os outros handlers, têm como objetivo buscar os dados da API, construir a interface para mostrar estes dados e permitir que o utilizador navegue entre várias páginas de resultados, sem ter de recarregar a página inteira.

# Estrutura de Navegação

A navegação da SPA é baseada em alterações do fragmento da URL (hash), permitindo simular múltiplas páginas sem recarregar o browser.  
O elemento central da navegação é o `window.location.hash`, que contém a rota atual, por exemplo:

- `#clubs` — lista todos os clubes.
- `#clubs/3` — mostra detalhes do clube com ID 3.
- `#courts/5` — mostra detalhes do court com ID 5.
- `#rentals` — lista as rentals do utilizador.

O `router.js` faz o parsing deste hash, identificando a rota e os parâmetros, invocando o handler correspondente para atualizar o conteúdo correto na página.

## Formato dos Parâmetros na Rota

As rotas suportam parâmetros posicionais indicados por chaves `{}`.  
Por exemplo:

- `clubs/{cid}` — onde `{cid}` é o ID do clube.
- `courts/{crid}` — onde `{crid}` é o ID do court.

Quando a hash muda, o router:

1. Extrai estes parâmetros da URL.
2. Passa-os para o handler associado, que pode então fazer as chamadas à API para obter os seus dados específicos.

**Exemplo:**  
Para a rota `#clubs/4`, o parâmetro `cid` terá o valor `4`.

## mainContent

O elemento HTML com `id="mainContent"` é o principal container onde o conteúdo dinâmico da SPA é renderizado, onde todos os handlers substituem o conteúdo deste elemento para mostrar a vista correspondente à rota atual.

Este padrão permite:

- Manter a estrutura da página (como cabeçalho e rodapé) fixa.
- Atualizar dinamicamente apenas o conteúdo principal, proporcionando uma experiência de navegação fluida e rápida.

- **`styles/common.css`**  
  Ficheiro com os estilos CSS comuns a toda a aplicação.

### Funcionalidade

A SPA consome os endpoints da API para obter e apresentar dados ao utilizador, observando que a interface é construída dinamicamente com JavaScript, através da manipulação direta do DOM (`document.createElement`, `replaceChildren`, etc.).

Além das operações básicas de leitura (GET), a SPA foi enriquecida com funcionalidades que permitem ao utilizador interagir de forma mais completa com o sistema, incluindo:

- Pesquisa de Clubes por Nome: permite ao utilizador pesquisar clubes pelo nome, enviando o termo como query parameter para o endpoint `/clubs?name=...`, com resultados paginados e atualizados dinamicamente.
- Visualização de Horas Disponíveis para um Court numa Data Específica: na página de detalhes de um court, o utilizador pode selecionar uma data para consultar as horas disponíveis para reserva, consumindo o endpoint `/clubs/{cid}/courts/{crid}/availablehours?date=...`.
- Busca de Rentals por Data no Detalhe do Court: permite listar as rentals de um court num dia específico, via endpoint `/clubs/{cid}/courts/{crid}/rentals?date=...`.
- Criação de Clubes, Courts e Rentals: formulários para criação de novos recursos, enviando requisições POST com autenticação via token, atualizando a interface com mensagens de sucesso ou erro.
- Atualização e Remoção de Rentals: funcionalidades para editar reservas existentes (via PUT) e excluir reservas (via DELETE), com confirmação do utilizador e feedback visual.

### Sistema de Routing

- Implementado manualmente com `hash-based routing`.
- Permite navegar entre vistas como `#clubs`, `#clubs/3`, `#courts/5`, entre outras.
- O `router.js` faz o parsing da hash da URL, identifica os parâmetros e invoca o handler correspondente.



### Interação com a API

- Utiliza `fetch()` para realizar chamadas HTTP.
- Inclui o token de autenticação nos headers, quando necessário (`Authorization: Bearer <token>`).
- Trata erros de rede e de resposta da API, exibindo mensagens adequadas ao utilizador.

### Paginação

- A listagem de clubes, courts e rentals implementa paginação com `limit` e `skip`, permitindo navegar entre páginas com botões “Previous” e “Next”.

### Vantagens da Abordagem

- **Leveza:** Sem dependência de frameworks pesados como React ou Angular.
- **Simplicidade:** Fácil de manter e expandir, com lógica dividida por módulos (`handlers`).
- **Modularidade:** Cada funcionalidade (clubes, courts, rentals, users) está isolada num ficheiro próprio.
- **Escalabilidade:** A nova arquitectura modular permite adicionar novas funcionalidades com facilidade.


## Benefícios da SPA

- **Experiência de utilizador fluida:** Navegação rápida e sem recarregamentos da página.
- **Modularidade:** Código organizado em *handlers* específicos para cada tipo de recurso.
- **Interacção directa com a API:** Utilização eficiente dos métodos HTTP para operações de criação, leitura, actualização e eliminação (CRUD).
- **Feedback imediato:** Apresentação de mensagens claras que indicam o sucesso ou falha das operações.
- **Escalabilidade:** Estrutura facilmente extensível para incluir novas funcionalidades e recursos no futuro.


### Limitações Atuais
-A interface ainda pode beneficiar de melhorias de usabilidade.

-Algumas operações poderiam ter validações e feedback mais robustos.
### Possibilidades Futuras

- Integração de autenticação persistente e mais avançada.
- Expansão do sistema de routing para rotas mais complexas e reutilização de componentes.
