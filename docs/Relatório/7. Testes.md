# Testes dos Handlers

A aplicação inclui um conjunto de testes para os handlers, localizada na pasta `test/`. Estes testes são essenciais para assegurar a segurança da SPA, validando o correto funcionamento das funções que interagem com a API .

## Vantagens da Implementação dos Testes

- **Deteção  de erros**  
  Os testes permitem identificar possivéis falhas no código antes que estas afetem o utilizador final, facilitando a manutenção da aplicação.

- **Confiança na implementação**  
  Ao garantir que os handlers se comportam conforme esperado, os testes asseguram a estabilidade das funcionalidades principais mesmo após alterações no código.

- **Facilidade de refatoração**  
  Com uma base de testes sólida, é possível melhorar e otimizar o código com maior segurança, sabendo que eventuais problemas serão rapidamente detetados.



---

Desta forma, a inclusão dos testes contribui significativamente para a robustez e sustentabilidade do projeto, tornando a SPA mais confiável e preparada para futuras expansões.

# Testes do Módulo Kotlin

O projeto também inclui um conjunto de testes para o módulo desenvolvido em Kotlin, que cobre a lógica e a interação com a API no backend. Estes testes são fundamentais para garantir a qualidade, a correção e a estabilidade do código Kotlin.

## Principais vantagens dos testes no módulo Kotlin

- **Garantia de qualidade**  
  Os testes asseguram que as funcionalidades implementadas no backend funcionam conforme o esperado, evitando regressões e erros em produção.

- **Facilitam a manutenção**  
  Com testes automatizados, é possível modificar e melhorar o código com maior segurança, sabendo que eventuais problemas serão rapidamente identificados.

- **Documentação do comportamento**  
  Os testes servem pra ajudar no desenvolvimento do projecto, mostrando como as funções e classes devem se comportar em diferentes cenários.


- **Redução de custos**  
  Detectar e corrigir erros durante o desenvolvimento é muito mais eficiente do que fazê-lo após o lançamento, reduzindo custos e trabalho.

Dessa forma, a inclusão dos testes no módulo Kotlin contribui significativamente para a robustez e sustentabilidade do sistema, garantindo que o backend suporte adequadamente as funcionalidades da aplicação.
