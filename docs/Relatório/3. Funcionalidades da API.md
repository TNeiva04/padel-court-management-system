### Domínio da Aplicação

O domínio da aplicação é composto por quatro entidades principais:

- **Utilizadores**
- **Clubes**
- **Courts**
- **Rentals**

Cada entidade é caracterizada por um conjunto de atributos e relações, permitindo a criação e gestão de clubes de padel, respetivos courts e rentals realizados pelos utilizadores.

### Funcionalidades da API

A API a desenvolver deverá fornecer operações para a gestão de:

- Utilizadores
- Clubs
- Courts
- Rentals

Deverá ainda suportar:

- Mecanismos de **autenticação e autorização**, com tokens para validar e restringir o acesso às operações permitidas.

### Tecnologias

A solução foi implementada em **Kotlin**, recorrendo às seguintes bibliotecas:

- `http4k` para o manuseamento de requisições HTTP.
- `kotlinx.serialization` para serialização e desserialização de dados.

Para o frontend, foi utilizado JavaScript para desenvolver a ‘interface’ web como uma Single 
Page Application (SPA), permitindo uma interação dinâmica e fluida com a API sem necessidade 
de recarregar a página. A SPA é construída com **JavaScript** , manipulando diretamente o DOM e 
utilizando o padrão de routing baseado em hash para navegação

Nesta fase inicial, os dados serão armazenados **em memória**, sendo exigida uma arquitetura:

- Modular
- Legível
- Orientada para a testabilidade

Ou numa base de dados relacional, como o PostgreSQL, que será utilizada nas fases seguintes do projeto.
